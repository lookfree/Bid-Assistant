## 1. 产品概述

### 1.1 产品定位
AI 编标助手是一款**面向投标场景的 AI 标书生产力工具**。用户上传招标文件后，由 AI 完成「读标 → 提纲 → 标书生成 → 标书审查 → 导出」的完整流程，把投标中最耗时、最易出错的环节交给 AI，将一份标书的制作周期从几天压缩到几十分钟。

### 1.2 一句话价值主张
> 上传招标文件，AI 帮你写完整份标书。

### 1.3 核心价值
| 价值点 | 说明 |
| --- | --- |
| 从几天到几十分钟 | 自动读标、生成提纲与正文，免去重复劳动 |
| 紧扣评分点不漏项 | 每个章节自动对应招标评分点，覆盖率一目了然 |
| 废标风险提前发现 | 资质、响应、格式逐项体检，导出前拦住低级失误 |

### 1.4 目标用户
| 人群 | 特征与诉求 |
| --- | --- |
| 个人投标用户 | 首次写标书也能快速上手，按需付费、零负担 |
| 中小投标团队 | 多项目并行，统一模板与知识沉淀，提效 |
| 标书代写从业者 | 批量接单、快速交付，专注打磨方案 |

## 2. 信息架构与导航

### 2.1 站点结构
```
公开区（未登录）
├─ 首页 /                官网/落地页：价值主张、流程、CTA
└─ 登录注册 /login        手机号 + 验证码

工具区（登录后，带左侧边栏布局）
├─ 智能编标
│  ├─ 新建标书 /upload      上传招标文件
│  ├─ 招标解读 /read        读标结果（双栏）
│  ├─ 提纲生成 /outline     标书大纲（双栏，可编辑）
│  ├─ 标书生成 /content     正文生成（三栏 + AI 对话 + 导出）
│  └─ 标书审查 /risk        废标风险审查 + 标书查重
└─ 我的
   ├─ 我的标书 /projects    项目列表
   └─ 会员中心 /membership  积分 / 套餐 / 充值
```

### 2.2 主流程
```
首页 → 登录/注册 → 新建标书(上传) → 招标解读 → 提纲生成
     → 标书生成(编辑/AI对话/导出) → 标书审查 → 我的标书
```

### 2.3 权限与可见性
- **会员套餐入口仅登录后可见**（位于工具区侧边栏「会员中心」），公开首页不暴露套餐价格。
- 工具区全部页面需登录态访问；未登录访问将引导至 `/login`，并可带 `redirect` 与 `reason` 参数。

## 3. 功能需求详述

### 3.1 首页（落地页 `/`）
**目标**：传达价值主张、降低尝试门槛、引导注册。

| 模块 | 内容 |
| --- | --- |
| 顶部导航 | Logo、登录、免费试用（不含会员套餐入口） |
| Hero | 标题、副文案、「免费上传招标文件」CTA、产品界面预览图 |
| 流程区 | 五步流程：上传 → 智能读标 → 生成提纲 → 生成标书 → 标书审查 |
| 价值点 | 三张卡片（提效 / 不漏项 / 风险前置） |
| 适用人群 | 个人 / 中小团队 / 代写从业者 |
| 底部 CTA | 「免费开始」+ 信任要素（加密、可升级等） |

### 3.2 登录 / 注册（`/login`）
- 手机号 + 6 位短信验证码登录，未注册手机号自动创建账号。
- 验证码 60 秒倒计时重发；需勾选《用户协议》《隐私政策》方可提交。
- 支持 `redirect`（登录后跳转）与 `reason`（来源提示语，如付费拦截）参数。
- 页脚展示价值点与「数据全程加密」信任标识。

**校验规则**：手机号 `^1\d{10}$`；验证码 6 位数字；协议必须勾选。

### 3.3 新建标书 / 上传（`/upload`）
- 支持点击或拖拽上传，格式 DOCX / PDF / XLSX，单文件 ≤ 50MB，可多选。
- 除招标文件外，支持上传工程量清单、技术要求附件、历史参考方案等多类文件以提升解析精度。
- 实时上传进度，逐文件状态（上传中 / 已就绪），可移除、可继续添加。
- 支持「从我的标书选择」复用已有项目。
- 全部文件就绪后激活「开始智能读标」→ 跳转 `/read`。
- 提示：新用户免费解析 1 份招标文件；数据加密、仅本人可见。

### 3.4 招标解读（`/read`）
**布局**：左侧招标文件原文，右侧分类解读结果（双栏联动）。

- **左栏**：招标文件原文，按章节分段展示，支持锚点定位、高亮当前章节。
- **右栏**：六大分类解读——项目概况、资格要求、商务条款、技术需求、评分标准、格式与红线。
  - 每个条目标注状态：
    - **可索引（found）**：可点击，定位到左侧原文对应章节。
    - **未发现（missing）**：招标文件中未明确，标注「未发现」提示（不提供补充编辑，仅提示关注）。
  - 废标风险项以「废标风险」标签强调（如资质、保证金、签章、不可偏离★项）。
- 顶部统计：已覆盖项数 / 未明确项数。
- 支持生成读标报告（idle → generating → ready 状态）。
- 右下角悬浮按钮：「已知悉，生成投标文件大纲」→ 跳转 `/outline`。

### 3.5 提纲生成（`/outline`）
**布局**：左侧招标文件原文，右侧投标大纲（双栏联动）。

- **标签切换**：技术标大纲 / 商务标大纲 / 标书全文。
  - 「标书全文」为技术标 + 商务标合并视图，技术标 / 商务标的改动同步反映到全文。
- **来源索引**：
  - **可索引**：可点击定位到左侧招标文件对应章节。
  - **新增**：无法从招标文件索引、由 AI 补充的条目标注「新增」。
- **可编辑**（增删改）：
  - 条目：行内编辑文字、删除、在章节末尾「添加子项」（新增项默认标「新增」）。
  - 章节：编辑标题、删除章节、「添加章节」。
  - 保存（回车 / 对勾）、取消（Esc / X）。
- 顶部统计：可索引数 / 新增数（随编辑实时更新）。
- 右下角悬浮按钮跳转 `/content`。

### 3.6 标书生成（`/content`）
**布局**：三栏 —— 左侧目录、中间可编辑正文、右侧 AI 对话。

- **左栏 · 目录**：依据提纲生成的章节列表。
  - 状态标记：已生成（绿勾）、待生成（橙色警示）、新增（蓝标）。
  - 点击任一章切换正文。
- **中栏 · 可编辑正文**：
  - 富文本编辑器，工具栏支持加粗、斜体、小标题、列表、**插入图片**。
  - 编辑自动保存；未生成章节显示空状态 + 「AI 生成本章正文」。
  - 底部：「重新生成」+ 自动保存提示。
- **右栏 · AI 对话**：
  - 针对**当前章节 / 某处内容**与 AI 对话进行修改。
  - 快捷指令：扩写本章 / 更正式 / 提炼要点 / 补充案例等。
  - 可折叠。
- **标书范围标签**：技术标 / 商务标 / 标书全文（全文为技术标 + 商务标合并，联动同步）。
- **导出文件**（右下角）：
  - 选择导出范围：技术文件 / 商务文件 / 标书全文。
  - 选择格式：Word / PDF。
  - 导出进度与完成状态；导出后出现「我的标书」入口（提示「导出后可在『我的标书』随时重新下载」）→ `/projects`。

### 3.7 标书审查（`/risk`）
**两个子功能（标签切换）**：

#### 3.7.1 废标风险审查
- 上传**招标文件**（Tender Doc）+ **投标文件**（Bid Doc）双区域。
- AI 自动识别废标风险与潜在问题。
- 输出：健康分、高 / 中风险项 + 整改建议、已通过项。
- 隐私说明：浏览器本地存储、不上传服务器、模型不训练、文件阅后即焚。

#### 3.7.2 标书查重
- 上传 2–3 份投标文件做雷同比对；可选上传招标文件以排除同源段落、降低误报。
- **查重策略**（描述随选中档位动态变化）：
  - 快速：速度优先，命中明显雷同段落，适合投标前快速自查。
  - 标准：速度与查全率均衡，覆盖常见雷同（推荐）。
  - 严格：查全率优先，可识破同义改写与小段拼接，耗时略长。
- **查重维度**（按需启用）：文本指纹（必选，MinHash·LSH + LCS）、图片指纹（dHash + aHash）、元数据指纹（SimHash）、招标基线扣除（需招标文件）。
- 运行后展示最高相似度与两两比对结果。

### 3.8 我的标书 / 项目（`/projects`）
- 项目卡片列表，支持搜索项目名称。
- 每张卡片：项目名、阶段标签、更新时间、截止提示、进度条。
- 阶段：读标中 / 提纲中 / 正文生成中 / 待审查 / 待导出 / 已交付，点击进入对应流程页继续编辑。
- 「新建标书」入口 → `/upload`。

### 3.9 会员中心（`/membership`）
见第 4 节「商业化与积分体系」。

## 4. 商业化与积分体系

### 4.1 计费模型
采用**积分制（Credits）**：所有 AI 操作按「篇幅 · 字数 · 功能」分档消耗积分。

- **免费体验版**：手机号注册赠送 **200 积分**，基本可完整体验全流程 **1 次**（废标风险审查、标书查重等付费模块除外）。
- **会员版**：按档位每月赠送积分额度。
- **单独积分充值**：积分用尽时，可选择「升级会员」或「单独购买积分」；单独购买单价高于会员赠送，以引导优先升级会员。

### 4.2 会员档位（免费体验版 + 5 级会员）
| 档位 | 月度积分 | 月付 | 年付 | 关键权益 |
| --- | --- | --- | --- | --- |
| 免费体验版 | 200（一次性） | ¥0 | — | 读标/提纲/标书生成、导出 1 次 |
| 基础会员 | 1,200 | ¥39 | ¥399 | 完整生成、导出、废标审查（标准） |
| 基础会员 Pro | 3,000 | ¥89 | ¥899 | 标书查重、全文深度生成、5 并行项目 |
| 基础会员 Plus | 6,000 | ¥159 | ¥1,599 | 全维度查重、逐章重写、15 并行、优先算力 |
| 高级会员 Pro（最受欢迎） | 12,000 | ¥299 | ¥2,999 | 团队协作 5 席、30 并行、招标基线查重、专属客服 |
| 高级会员 Plus | 30,000 | ¥699 | ¥6,999 | 团队 20 席、不限并行、最高优先级、客户成功经理 |

### 4.3 渐进式套餐展示（核心规则）
- 「选择套餐」区域**仅展示两张卡片：当前档 + 下一档**。
- 用户升级到下一档后，卡片自动后移，露出再下一档方案。
- 已达最高档时提示「已是最高档」。
- 目的：降低决策复杂度，避免一次性呈现全部 6 档造成选择负担。

| --- | --- | --- |
| 招标解读 | 识别评分点与关键条款 | 20 积分 / 份 |

| 标书生成（短篇） | 单章 ≤ 2000 字 | 40 积分 / 章 |

| 废标风险审查 | 全文风险体检 + 整改建议 | 60 积分 / 次 |
| 标书查重 | 多维指纹比对 | 100 积分 / 次 |
| 导出 Word / PDF | 整本投标文件导出 | 20 积分 / 次 |

### 4.5 单独积分充值包
| 充值包 | 积分 | 价格 | 单价 |
| --- | --- | --- | --- |
| 入门包 | 500 | ¥29 | ¥5.8 / 100 积分 |

### 4.6 会员中心页面构成

## 5. 非功能性需求

| 类别 | 要求 |
| --- | --- |
| 数据安全 | 文件全程加密传输与存储，仅本人可见；审查场景支持本地存储 / 不上传 / 不训练 / 阅后即焚 |
| 性能 | 上传秒级反馈；生成类操作提供进度与状态反馈，长任务可后台进行 |
| 可访问性 | 语义化结构、ARIA 标签、键盘可操作、充分对比度 |
| 响应式 | 移动优先，自适应到桌面三栏布局 |
| 一致性 | 统一蓝紫渐变品牌色与设计令牌，组件风格一致 |

## 6. 设计规范要点
- **品牌色**：蓝 → 紫渐变（gradient-brand）作为主行动色，辅以中性灰与成功 / 警示语义色。
- **字体**：单一无衬线字体家族，标题加粗、正文 1.4–1.6 行高。
- **布局**：Flexbox 优先；工具页采用左侧边栏 + 内容区；解读 / 提纲为双栏，标书生成为三栏。
- **图标**：lucide-react，统一尺寸（16/20/24）。
- 通用文案统一为「标书生成 / 标书审查」，导出能力整合进标书生成页。

## 7. 关键流程状态机

### 7.1 项目阶段
```
读标中 → 提纲中 → 正文生成中 → 待审查 → 待导出 → 已交付
```

### 7.2 积分扣减判定（建议实现）
```
触发付费操作
  ├─ 余额充足 → 扣减积分 → 执行
  └─ 余额不足 → 提示：升级会员 / 单独充值积分
```
