# spec330 · 标书生成配置（目标字数/页数 + 输出格式）实现计划

**Goal:** 点「开始生成正文」时弹出**生成配置**：
① 目标体量——滑杆选标书总字数,实时换算约多少页（参考用户提供的竞品样式:滑杆 + 右侧「约 N 万字 / N 页」+ 自定义字数勾选）;生成按此**规划各章篇幅**。
② 输出格式——正文/标题字体字号、行距、页边距可配置;导出 docx 按此渲染。

## UI 布局（生成确认弹层,两段式）

```
┌ 生成配置 ─────────────────────────────────┐
│ ▍选择标书字数                              │
│ ●────────────────○──   约 12.6万字 · 210页 │   ← 滑杆 1~50万字,页数=字数/600(A4 口径,与站内字数统计一致)
│ ☐ 自定义标书字数  [ 126000 ]               │   ← 勾选后滑杆禁用,直接输数
│ 建议:按本标书 24 章,推荐 7~15 万字         │   ← 默认值=章节数×3000,夹在滑杆范围内
│ ─────────────────────────────────────────  │
│ ▾ 输出格式（可折叠,默认收起,记住上次选择） │
│   正文字体 [宋体 ▾]     正文字号 [小四 ▾]   │
│   标题字体 [黑体 ▾]     章标题字号 [三号 ▾]  │
│   行距 [1.5 倍 ▾]       页边距 [常规 ▾]     │
│   预设:〔政府采购常用〕〔紧凑〕〔默认〕      │   ← 一键填充上面六项
│ ─────────────────────────────────────────  │
│ 预计消耗:content_long 档 N 积分   [开始生成] │   ← 目标字数决定预计档位,明示后再扣
└───────────────────────────────────────────┘
```

- 格式作用于**导出 docx**（编辑器屏显样式不变——屏幕排版与打印排版分离,避免编辑器跟着字体设置漂移）。
- 选择持久化到 localStorage（用户级偏好,下次默认带出）;不入库不加表。

## 链路设计

- **web**：content 页「开始生成」按钮 → 生成配置弹层（新组件 generation-config.tsx,含滑杆/自定义/格式面板/预设）→
  确认后 `start({ target_chars, format })`;use-step 的 start 支持透传 run_input 补充键。
  导出时把 format 一并写入 export 步 run_input（`use-export` 从 localStorage 读同一份偏好）。
- **api**：`POST /:id/steps/content` 与 `/:id/steps/export` 的 runInput 白名单放行 `target_chars`(int,1万~50万夹住)
  与 `format`(六键枚举,zod 校验,坏值 400);随 run_input 下发 agent,**不动计费逻辑**（content 分档结算按实际产出
  maxChapterChars,既有 settleContent 不变——目标字数只影响"预计"文案）。
- **agent content 节点**：规划轮把 `target_chars` 按章节权重（评分权重高的章多分）拆成**各章字数预算**,
  写手 prompt 注入「本章目标约 N 字（±20%）」;缺省（不传）行为与今天逐字节一致。
- **agent export 渲染**：`render_docx` 接受 format——`_apply_bid_styles` 按配置设置正文/标题字体字号
  （宋体/仿宋/楷体/黑体;小四=12pt/四号=14pt/五号=10.5pt;三号=16pt 等 GB 字号映射）、行距（单倍/1.5/固定22磅）、
  页边距（常规2.54/3.18cm、窄1.27/1.27、宽3.18/3.81）;缺省=现行样式逐字节不变。
  中文字体用 `w:eastAsia` 设置（python-docx 需 rPr 补 eastAsia 才对中文生效——现有 _apply_bid_styles 已有先例）。

## 验收

- 选 10 万字 → 生成的全书总字数在目标 ±25% 内,章节间按权重分配（权重高的章明显更长）;
- 不动配置直接生成 → 与今天行为一致（缺省即现状）;
- 配置仿宋/小四/1.5倍行距/窄边距导出 → Word 里逐项生效;缺省导出与现有文件版式一致;
- 预计消耗档位与实际结算档一致口径展示;坏参数 400 不扣费不占步位。
