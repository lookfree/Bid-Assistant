## 三、系统功能缺口(需开发,按开发量从小到大)

### 1. AI 生成内容显式提示标识(图5/图6,报告§一(五))

三处全缺:

- C 端五个结果页(read/outline/content/risk/present)无"算法辅助生成、请人工复核"标识;
- 导出确认弹窗 `apps/web/app/(tool)/content/export-menu.tsx` 只有格式选择与积分预估;
- 导出文件:`services/agent/.../render/docx.py` `_add_page_number_footer` 页脚只有页码,pptx 亦无说明页。

报告已给定文案,照抄:"本内容由智启元投标助手生成合成类算法辅助生成,仅供投标文件编制参考,请结合招标文件原文和企业实际情况复核确认后使用。"要求:页面标识系统自动添加、**用户不可关闭**;导出文件由导出环节自动写入。

### 2. 服务协议/隐私政策页面(图13)

登录页已有强制勾选与拦截(`apps/web/app/login/page.tsx:302-308`),但《用户协议》《隐私政策》链接为 `href="#"` 死链,全站无 `/terms`、`/privacy` 页面。主要工作量是协议文本撰写(个人信息处理、第三方共享、用户权利三块必须覆盖,与报告"用户个人信息保护"节对齐),页面本身很薄。

### 3. 算法公示页(图17)

不存在。需一个静态页:算法名称(智启元投标助手生成合成类算法)、机制机理、应用场景、目的、申诉渠道。内容以报告"拟公示内容"附件为准(附件本身也待写,见四.1)。上线后此页 URL 即备案要求的"公示 URL"。

### 4. 投诉/客服/在线反馈入口(图12)

- C 端唯一的"联系客服"在 `apps/web/app/(tool)/membership/page.tsx:389`,`<Link href="/">` 指向首页,是死链;
- `apps/api` 无 feedback/complaint 路由;
- 后台 `support` 只是 RBAC 角色名,无工单模块。

报告要求申诉记录可追溯(受理时间、问题类型、项目/任务标识、处理过程、处理结论、合理期限反馈)。最简实现:C 端反馈表单 + `feedback` 表(user_id/类型/项目/run_id/内容/状态/处理记录) + admin 查看处理页。

### 5. 敏感词/内容安全机制(图14 前半)

`apps/api`、`services/agent` 搜敏感词/moderation/内容安全零命中,目前完全依赖第三方模型平台侧审核(`07-审核与风控规则说明.md` 已标"待补充:自建敏感词库与拦截策略")。最简实现:输出侧敏感词库检测 + 命中记录(可复用 agent 观测表或独立表),供图14 截图与"违规输出上报处置"叙述使用。

### 6. 违规样本台账/人工审核记录(图14 后半/图15)

后台无模块、`apps/api/src/db/schema/` 无相关表。**可不开发**:备案层面接受线下台账(Excel/文档,记录样本来源、触发步骤、模型、问题类型、处置结果)+ 定期抽检表,截图台账文档同样成立。若后续做成系统功能,挂在 admin 下与三.4 的反馈处理合并考虑。
