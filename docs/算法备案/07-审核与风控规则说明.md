# 审核与风控规则说明

## 一、输出内容安全规则

### 渲染清洗(强制,双入口)

模型生成的章节 HTML 在**收稿入口**与**渲染入口**统一经过清洗(幂等),剥离:

| 剥离对象 | 风险 |
|---|---|
| `<script>` | 前端脚本注入 |
| `<style>` / `<head>` | 样式劫持全页布局 |
| `<!DOCTYPE>` / `<html>` / `<body>` 文档壳 | 破坏渲染结构、导出污染 |
| `<meta>` / `<title>` | 同上 |

### 结构化输出校验

读标/提纲/述标强制模型走工具调用返回 JSON,服务端 schema 校验不通过即重试/失败,不把畸形输出交付用户。

### 内容安全依赖

- 第三方大模型供应商平台侧内容安全(供应商均为已备案生成式 AI 服务)。
- 【待补充:自建敏感词库与拦截策略、违规输出上报处置流程】。

## 二、标书审查算法的定位(审核辅助)

- 审查步骤输出**风险项清单**(风险等级+原文定位+修改建议),**仅供用户人工复核参考**;系统不做自动处置、不拦截用户文档。
- 查重引擎(自研文本相似度规则)辅助识别正文重复段落,同样仅提示。
- 投标文件最终由用户人工确认后自行使用,算法不代替人做投标决策。

## 三、业务风控规则

### 计费风控(资金红线)

| 规则 | 说明 |
|---|---|
| 显式触发 | 计费步骤只由用户点击触发;`autostart` 参数一次性消费后立即从 URL 摘除,防重放重复扣费 |
| 幂等 | 每步 hold→settle/release 全幂等;每笔变更带幂等键 |
| 失败必退 | 生成失败自动全额退还(生产验证:单日 8 次失败笔笔净 0) |
| 账本 | 余额 = append-only 账本求和;多来源积分按过期时间 FIFO |
| 资金隔离 | 智能体服务 money-blind,只报用量 |

### 访问风控

| 规则 | 说明 |
|---|---|
| 登录 | 短信验证码;发送冷却 + 尝试次数限制(Redis 计数) |
| 会话 | C 端 7 天、运营后台 8 小时,独立身份体系 |
| 步骤顺序 | 前置步骤未完成返回 409,前端渲染引导卡而非计费按钮 |
| 前端计费 CTA | 只在数据确认态渲染(加载中/错误态不亮),防误触 |

### 稳定性风控

| 规则 | 说明 |
|---|---|
| 模型三层超时 | 空闲 30s / 单轮 1200s / 瞬断因果链识别 → 降级链重试 |
| 未配置拒绝 | 模型配置缺失 → 400,不预扣不静默回退 |
| 任务自愈 | 卡死任务超阈值自动标失败+退款(幂等) |
| RAG 降级 | 嵌入服务/向量库故障 → 检索返回空,生成不受阻 |
| 部署管控 | 发版前强制检查在途任务(`status='running'`),避免打断用户生成 |
