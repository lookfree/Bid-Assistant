# 模型说明文档

## 一、大语言模型(第三方 API,纯推理)

### 供应商与型号

系统支持接入以下国内大模型供应商(均为已完成生成式 AI 服务备案的供应商);**实际启用的具体型号由运营后台「模型编排」配置决定**,代码中不写死任何默认模型:

| 供应商 | 系列 | 调用方式 |
|---|---|---|
| DeepSeek(深度求索) | deepseek 系列 | OpenAI 兼容 HTTPS API |
| 智谱 AI | GLM 系列 | OpenAI 兼容 HTTPS API |
| 阿里云 | 通义千问 Qwen 系列 | OpenAI 兼容 HTTPS API |

### 编排机制

- 每个生成步骤配置**主模型 + 降级链**(1~N 个备选);主模型失败/超时自动按链切换。
- **未配置模型 → 拒绝生成**(400 `model_not_configured`),不占步骤名额、不预扣积分、不静默回退任何默认模型——运营后台移除配置即可全局停用生成能力。
- **思考模式(深度推理)默认关闭**,每模型可单独开启;开启思考的模型走非流式调用(与流式强制结构化输出不兼容),关闭思考的按 provider 下发对应关闭参数。

### 调用韧性(每次调用统一生效)

| 层 | 参数 | 作用 |
|---|---|---|
| 空闲超时 | 30s 无新 token | 识别挂死连接 |
| 单轮总时长 | 1200s 上限 | 识别限流下的"慢而不死"细流 |
| 瞬断识别 | 沿异常因果链下钻匹配 | 网络瞬断进入重试通道而非直接失败 |
| 降级重试 | 按配置链切换 | 供应商级故障隔离 |

### 结构化输出

读标/提纲/述标等步骤强制模型以工具调用(tool_choice)返回结构化 JSON,服务端按 schema 校验;正文步骤输出 HTML 并经强制清洗(见 07 号文档)。

## 二、嵌入模型(开源,自部署)

| 项目 | 内容 |
|---|---|
| 模型 | BGE-M3(开源) |
| 用途 | 文本向量化(1024 维 dense),仅用于 RAG 相似度检索 |
| 部署 | 自部署独立 HTTP 服务(OpenAI 兼容 `/v1/embeddings`),CPU 推理,不对公网暴露 |
| 数据 | 输入文本不出自有环境;仅推理,不训练 |
| 降级 | 服务探活失败 → RAG 整体禁用,生成主流程照常(无参考资料段) |

## 三、向量检索

| 项目 | 内容 |
|---|---|
| 存储 | PostgreSQL + pgvector(`agent.rag_chunks`,vector(1024)) |
| 检索 | cosine 近邻;资料库(library)走 HNSW 近似索引(partial index),招标条款(tender)限定单项目精确检索 |
| 隔离 | 强制 `user_id` 过滤;tender 再按项目(thread_id)隔离 |
| 降级 | 单次检索 2s statement_timeout,超时返回空,不阻塞生成 |
| 生命周期 | 项目向量 30 天不活跃自动清扫;资料库向量随资料删除即删 |

## 四、训练/微调声明

本系统**不进行任何模型训练或微调**:大语言模型为第三方 API 纯推理调用;嵌入模型为开源模型原样推理部署;**任何用户数据均不用于模型训练**(包括不向第三方供应商提供训练授权,API 调用均为推理接口)。
