# 权限控制说明

## 一、身份体系(双轨隔离)

| 体系 | 用户 | 认证方式 | 会话 |
|---|---|---|---|
| C 端(app.) | 投标用户 | 手机号+短信验证码(发送冷却、尝试次数限制) | 7 天(AUTH_SESSION_TTL_DAYS) |
| 运营后台(admin.) | 运营人员 | 独立账号体系 | 8 小时(刻意短于 C 端) |

两套体系独立子域、独立会话签发,互不通用;C 端凭据无法访问任何运营接口,反之亦然。

## 二、数据级权限

| 资源 | 控制 |
|---|---|
| 项目/步骤/结果 | 全部按 user_id 归属校验,只能访问自己的项目 |
| 资料库 | CRUD 与 RAG 索引删除均带属主校验(user_id 不匹配的行不受影响) |
| 向量检索 | SQL 层强制 `user_id` 过滤 + tender 按项目隔离,不存在跨用户检索路径 |
| 上传/下载 | MinIO 预签名 URL,按用户按对象签发,限时有效 |
| 积分账本 | 用户仅可见自己的账单;变更仅 App API 服务端发起 |

## 三、服务间权限

| 边界 | 控制 |
|---|---|
| 智能体服务 | 不对公网暴露,仅 App API 经内网调用;对用户仅持有 user_id 隔离键,不掌握手机号/资金(money-blind) |
| 嵌入服务 | 仅内网访问,无认证信息、无用户标识,只收发文本与向量 |
| 数据库 | 业务账号无 CREATE EXTENSION 等高危权限,schema 级隔离(public/langgraph/agent) |
| 第三方 API 凭据 | 只存服务器本地 `.env` 文件(gitignore),不入代码库;代码库无任何真实密钥 |

## 四、运维权限

- 服务器 SSH 仅授权人员密钥登录。
- 生产数据库直连凭据仅运维/研发核心人员持有。
- 运营后台权限:模型编排、计费配置、用户/订单查询;**运营后台移除模型配置即可停用生成算法**(应急开关)。
