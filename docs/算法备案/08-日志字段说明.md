# 日志字段说明

## 一、智能体观测日志(PostgreSQL `agent` schema,四表)

### agent_request(每次生成任务一行)

| 字段 | 说明 |
|---|---|
| run_id | 任务唯一标识 |
| thread_id | 项目会话标识(一个项目一个 thread,可跨 run 追溯) |
| agent_type | 算法标识(`bidding_agent`) |
| user_id | 用户隔离键(不含手机号等身份信息) |
| status / error | 任务状态与错误摘要 |
| created_at / finished_at | 起止时间 |

### agent_event_log(任务内事件明细)

| 字段 | 说明 |
|---|---|
| run_id / thread_id | 关联键 |
| event_type | 事件类型(节点开始/结束、模型调用、模型错误、降级切换等) |
| payload | 事件详情(JSON;含错误堆栈摘要,不含用户文档全文) |
| created_at | 时间戳 |

### agent_token_usage(每次模型调用一行——计费与审计双依据)

| 字段 | 说明 |
|---|---|
| run_id / thread_id / node | 调用位置(哪一步哪个节点) |
| provider / model | 实际使用的供应商与型号(含降级后) |
| prompt_tokens / completion_tokens | 用量 |
| latency_ms / finish_reason | 延迟与结束原因 |

### agent_tool_call(模型工具调用明细)

| 字段 | 说明 |
|---|---|
| run_id / tool_name / duration_ms / ok | 工具调用行为审计 |

## 二、App 业务日志(PostgreSQL `public` schema)

| 表 | 关键字段 | 用途 |
|---|---|---|
| project_steps | project_id / step / run_id / status / cost_points / created_at | 每次算法调用的业务侧记录(与 run_id 对应) |
| credit_transactions | 幂等键 / 类型 / 金额 / 余额快照 / created_at | append-only 积分账本(永久保留) |
| 登录/会话 | 手机号(脱敏展示)/ 登录时间 / IP | 账号安全审计 |

## 三、运行日志

- 容器 stdout/stderr(Docker 日志):服务启动、warning 级以上异常。
- Redis 进度流(`bid:` 前缀):任务实时进度事件,stream 上限 1000 条自动裁剪,属临时数据。

## 四、留存与访问

| 项 | 现状 | 建议留存策略 |
|---|---|---|
| 四张观测表 | 数据库长期保存(未设自动清理) | ≥6 个月(满足算法调用可追溯要求) |
| 积分账本 | 永久(append-only,不可篡改语义) | 永久 |
| 业务步骤记录 | 随项目保留 | 随项目 |
| Redis 进度/缓存 | 24h TTL / stream 裁剪 | 临时数据,无需长留 |
| Docker 运行日志 | 随容器滚动 | 建议接入集中收集后 ≥6 个月 |

**访问控制**:数据库凭据仅授权运维/研发持有(密钥只存服务器本地 env 文件,不入代码库);运营后台可见脱敏后的用量统计与账本;普通用户仅可见自己项目的进度与结果。
