# 系统架构图

> 三层架构,资金与算法严格隔离:所有资金/鉴权在 App API 层;智能体服务只做生成,不接触资金(money-blind)。

```mermaid
flowchart TB
    subgraph 用户侧
        U1[C 端用户浏览器<br/>app. 投标助手]
        U2[运营人员浏览器<br/>admin. 运营后台]
    end

    subgraph 自有服务器["自有服务器(Docker Compose)"]
        NG[Nginx 反向代理]
        WEB[Web 前端<br/>Next.js]
        ADM[运营后台前端<br/>Next.js]
        API["App API(Hono/Bun)<br/>鉴权·计费·业务编排<br/>【资金唯一入口】"]
        AGT["智能体服务 API(FastAPI)<br/>生成任务受理·SSE·RAG索引"]
        WRK["智能体 Worker<br/>LangGraph 工作流执行"]
    end

    subgraph 数据层
        PG[("PostgreSQL<br/>public:业务+积分账本<br/>langgraph:工作流检查点<br/>agent:观测日志+向量库")]
        RD[("Redis<br/>会话·队列·进度流·续跑缓存")]
        MIO[("MinIO 对象存储<br/>招标文件原件·导出文件")]
    end

    subgraph 自部署模型
        EMB["BGE-M3 嵌入服务<br/>(开源模型,仅推理)"]
    end

    subgraph 第三方服务["第三方服务(仅 API 调用)"]
        LLM["大模型 API<br/>DeepSeek / 智谱GLM / 通义Qwen<br/>(运营后台配置主+降级链)"]
        SMS[阿里云短信]
        PAY[收钱吧支付]
    end

    U1 --> NG --> WEB --> API
    U2 --> NG --> ADM --> API
    API --> PG & RD & MIO
    API -->|创建 run / SSE 转发| AGT
    AGT --> RD
    WRK -->|消费任务队列| RD
    WRK --> PG & MIO
    WRK -->|推理调用| LLM
    WRK & AGT -->|向量化| EMB
    API --> SMS & PAY
```

## 关键边界

| 边界 | 说明 |
|---|---|
| 资金边界(铁律) | 积分/支付的一切变更只发生在 App API;智能体服务仅上报 token 用量,永不接触资金;每笔扣减/回调带幂等键;余额=append-only 账本求和 |
| 身份边界 | C 端与运营后台独立子域、独立会话体系(C 端 7 天,后台 8 小时) |
| 网络边界 | 智能体服务与嵌入服务不对公网暴露,仅经内部网络访问;对外仅 Nginx 80/443 |
| 数据边界 | 所有向量检索/文档访问按 user_id 强制过滤;跨用户不可见 |
| 模型边界 | 模型型号唯一来源=运营后台配置;未配置 → 拒绝生成(400),不静默回退默认模型 |
