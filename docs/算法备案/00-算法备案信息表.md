## 2. 应用位置

- **系统**:投标助手 SaaS(Web 应用)。C 端用户站点(app.)与运营后台(admin.)为独立子域、独立身份体系。
- **页面/业务流程**:C 端「工具流」页面,按固定管线顺序调用:上传 → **智能读标** → **提纲生成** → **正文生成** → **标书审查** → **述标演示** → 导出。每一步由用户**显式点击**触发,不存在自动/后台代用户发起的生成。
- **接口**:App API(`/projects/:id/steps/*` 等,见 04-接口说明);App API 再调用内部智能体服务(`/agents/bidding_agent/runs`)。智能体服务不对公网暴露。

## 3. 输入数据

| 类别 | 内容 | 说明 |
|---|---|---|
| 用户上传文档 | 招标文件(PDF/Word/多文件包) | 解析为纯文本条款后作为模型输入;原件存对象存储(MinIO) |
| 用户资料库 | 用户自有的企业资质、业绩等参考文档 | 仅用于该用户自己的 RAG 检索,按 user_id 严格隔离 |
| 用户指令 | 步骤点击、章节重写指令、编辑内容 | — |
| 业务规则 | 运营后台配置的模型编排、计费口径 | — |
| **不采集** | 用户行为画像、设备指纹、第三方数据 | 本算法无推荐场景,不构建用户画像 |

## 4. 输出结果

| 步骤 | 输出 | 形式 |
|---|---|---|
| 读标 | 招标文件结构化解读(基础信息/评分办法/技术需求/包件划分),**每条带原文条款引用(clause_id)** | 结构化 JSON,前端可视化 |
| 提纲 | 投标文件章节提纲 | 结构化 JSON |
| 正文 | 投标文件正文(逐章生成) | HTML(经清洗)→ 可导出 docx |
| 审查 | 合规风险项清单(风险等级+定位+建议) | 结构化报告,**仅供人工复核参考** |
| 述标 | 演示 PPT 结构 + 口播稿 + 问答 | 结构化 JSON |

## 5. 算法流程(概要)

文档解析(格式转换/条款分块)→ 分段并行 LLM 结构化提取(读标)→ 提纲生成 → 逐章正文生成(生成时以 RAG 检索结果注入「参考资料」段)→ 审查 → 渲染清洗 → 导出。详见 **02-算法流程图**、**03-数据流向图**。

## 6. 技术路线

- **大语言模型(第三方 API,纯推理)**:所有文本生成/提取环节。
- **RAG 向量检索**:自建 PostgreSQL + pgvector 向量库;嵌入模型为开源 BGE-M3(自部署,仅推理);cosine 近邻检索。
- **规则引擎**:输出渲染清洗(剥离 script/style 等,防注入)、文本相似度查重(自研规则)、业务校验(步骤顺序、幂等计费)。
- **无**:自训模型、深度学习训练管线、推荐/排序模型。

## 8. 训练/微调情况

**不训练、不微调。** 全部模型为纯推理调用;**不使用任何用户数据进行模型训练**;无训练数据集、无训练周期、无训练评估指标(不适用)。

## 9. 部署方式

- 自有服务器 Docker Compose 部署(Web 前端 / 运营后台 / App API / 智能体服务 / Nginx)。
- 数据库:云 PostgreSQL(业务库 + 向量库 + 智能体观测库,三 schema 隔离);Redis(缓存、任务队列、进度流);MinIO(对象存储)。
- 第三方大模型仅 API 调用,不在本地部署;嵌入服务 BGE-M3 自部署。
- 异步机制:生成任务经 Redis 队列由 worker 进程消费;进度经 SSE 实时推送;RAG 索引为后台异步任务;读标分段结果有断点续跑缓存(Redis,24h TTL)。

## 11. 安全措施

- **输出安全**:渲染层强制清洗(剥离 script/style/文档壳,防前端注入);生成内容仅面向用户自己的投标场景,不公开传播。
- **内容安全**:第三方大模型供应商平台侧内容安全机制(供应商均为已备案服务);【待补充:自建敏感词库/内容审核接入计划】。
- **访问安全**:短信验证码登录(冷却+尝试次数限制)、C 端/后台独立身份、会话管理(C 端 7 天/后台 8 小时)。
- **数据隔离**:所有检索与生成严格按 user_id 隔离,用户间数据不可互见。
- **稳定性**:模型三层超时(空闲 30s/单轮 1200s 上限/降级链重试)、限流退避、异常自愈(卡死任务自动标失败+退款)。
- **人工复核**:审查结论仅供参考,投标文件最终由用户人工确认后使用。

详见 **07-审核与风控规则说明**。

## 14. 应急机制

| 场景 | 机制 |
|---|---|
| 模型 API 故障/超限 | 主模型 → 降级链自动切换;三层超时判定;瞬断沿因果链识别后重试 |
| 模型未配置/需下线 | 运营后台移除模型配置 → 服务端拒绝生成(400),**等效一键关闭算法**,不静默回退 |
| 违规/异常输出 | 渲染清洗兜底;用户可重新生成;【待补充:违规输出上报处置流程】 |
| 检索故障 | RAG 降级为空(2s 超时),生成主流程不受阻 |
| 任务卡死 | 自愈机制(超阈值自动标失败+全额退款,幂等) |
| 发版风险 | 部署前强制检查在途任务,避免打断用户生成 |
